# go-trader Project Context

## Environment
- Go 1.26.2 (`brew install go@1.26` on macOS); use full binary path if not on PATH (e.g. `/opt/homebrew/bin/go`).
- Python runs through `uv run --no-sync python` for dev/backtest/manual CLI use; Go subprocess calls (scheduler) use `.venv/bin/python3` directly — deterministic relative path after `uv sync`, no PATH config needed in systemd.
- Git worktrees work without a copied virtualenv; run `uv sync` once per checkout.
- **systemd:** `go-trader.service` and `systemd/go-trader@.service` use `ProtectSystem=strict`; no `PATH`/`UV_CACHE_DIR` env injection needed since scheduler uses `.venv/bin/python3`. `/opt/go-trader/.env` (or per-instance `.env`) can set `DISCORD_BOT_TOKEN` and other secrets.

## Setup
- `uv sync` — install Python deps
- Copy `scheduler/config.example.json` → `scheduler/config.json`, fill API keys
- New server: `install https://github.com/richkuo/go-trader and init`

## Decision Priorities
- **Pick the best technical solution.** Don't down-scope to avoid test churn, large diffs, or long implementation time. Those are not constraints on design quality.
- **Tests are not a limiting factor for design.** If the right implementation breaks existing tests, that is acceptable — update or rewrite the tests after the feature is complete. Do not contort a design to keep tests passing.
- **Code volume is not a constraint.** Don't shrink scope to minimize lines changed. If 2000 lines is the right answer, write 2000 lines.
- **Time-to-implement is not a constraint.** Don't pick a faster-but-worse approach to "save time." Pick the correct approach.
- These rules govern *quality of the chosen solution*, not *scope of the task*. They don't override the branch+PR workflow, the "verify issue claims against code" rule, or destructive-action safety rules.

## Key Patterns
- Run git from repo root. Prefer `go -C scheduler build .` over `cd scheduler &&`.
- **New platform:** (1) `platforms/<name>/adapter.py` + `__init__.py`, (2) `shared_scripts/check_<name>.py`, (3) `executor.go`, (4) `config.go` (prefix + validation), (5) `fees.go`, (6) `main.go` dispatch, (7) `init.go` wizard + `generateConfig`, (8) `pyproject.toml`.
- **New options-on-existing-platform:** extend adapter with `get_vol_metrics`/`get_real_expiry`/`get_real_strike`/`get_premium_and_greeks`; add to `check_options.py` + `CalculateOptionFee` + init `OptionPlatforms`.
- Adapters via `importlib`; class detected by `endswith("ExchangeAdapter")` (one/file). Check scripts use public methods only.
- **Close registry import:** never `import registry` directly (collides with open registry). Use `from close_registry_loader import evaluate, list_strategies, build_close_registry` (in `shared_tools/`); loads via `importlib.util.spec_from_file_location` and caches.
- Subprocess contract: scripts emit JSON to stdout (even on error); exit 1 on error; Go parses regardless of code.
- **State locking:** `mu sync.RWMutex`. 6-phase loop: RLock → Lock(CheckRisk) → no-lock(subprocess) → Lock(execute) → marks → RLock(status). Audit: `grep -n "mu\.\(R\)\?Lock\(\)" scheduler/main.go`.
- Platform dispatch: use `s.Platform == "ibkr"`, never ID prefix. ID prefix map: `hl-`/`ibkr-`/`deribit-`/`ts-`/`rh-`/`okx-`/`luno-`, else BinanceUS.
- Strategy types: `spot`, `options`, `perps`, `futures`, `manual`. Perps paper reuses `ExecuteSpotSignal`; live calls `RunHyperliquidExecute` first. Futures use `ExecuteFuturesSignal`. **Manual** has no script/interval — `LoadConfig` auto-fills `script`/`args[0]="hold"`/`interval_seconds`/SL@1.5×ATR + `tiered_tp_atr_live`; scheduler runs close-eval, enqueues into `pending_manual_actions`. `CheckRisk` skipped. May share a coin with manual/HL perps peers — owner guards prevent cross-strategy mutation; full-close uses `shouldCloseFullPosition` + `extraCancelOIDs`.
- **Open/close split:** `OpenStrategy` overrides entry; `CloseStrategies` ordered, max `close_fraction` wins; empty → open-as-close. `strategy_composition.{go,py}`. Go emits `--strategy-refs <json>`; Python `parse_strategy_refs_arg` + `evaluate_open_close`. Duplicate close-ref names: last wins. Close before open. Options not split. `Regime` propagated. **Partial-close:** `close_fraction` decrements `pos.Quantity`, preserves `InitialQuantity`; SL cancel/replace suppressed. **Position-aware close evaluators:** `--position-{side,avg-cost,qty,initial-qty,entry-atr,regime}` (`PositionCtx`); `strip_unsupported_position_context` filters keys — **VAR_KEYWORD wrappers don't short-circuit:** `def *_strategy(df, **params)` must name framework kwargs explicitly. Cache key includes merged params.
- **Live exec guard:** every dispatch in main.go uses `liveExecFailed`; when `runXxxExecuteOrder` returns `ok2=false`, set true and skip state update.
- **Skip-reason guards:** `runHyperliquidExecuteOrder` / `runOKXExecuteOrder` / `runRobinhoodExecuteOrder` / `runTopStepExecuteOrder` must check the same conditions as `ExecuteXxxSignal` BEFORE spawning — otherwise on-chain fills land with no Trade record. Use `PerpsOrderSkipReason` / `SpotOrderSkipReason` / `FuturesOrderSkipReason`. Capture `posSide` alongside `posQty` in Phase 1 RLock — `Position.Quantity` is always positive.
- **Bidirectional perps:** `Direction` enum (`"long"`|`"short"`|`"both"`). Short-entry strategies (`triple_ema_bidir`, `bear_pullback_st`, `vwap_rejection_st`, `donchian_breakout`, `chart_pattern`, `liquidity_sweeps`) must be in `bidirectionalPerpsStrategies` (init.go). `ExecutePerpsSignalWithLeverage`+`PerpsOrderSkipReason` thread direction. Flip sizing: `perpsLiveOrderSize`; catastrophic flip degrades to close-only. `ValidatePerpsDirectionConfig` flags gap classes.
- `dueStrategies` is value-copied from `cfg.Strategies` — mutations don't persist; update `cfg.Strategies` first.
- Notification features: add methods to `MultiNotifier`; don't access `backends`.
- HL sys.path: SDK `hyperliquid` clashes with `platforms/hyperliquid/` — add `platforms/hyperliquid/` directly to sys.path, then `from adapter import HyperliquidExchangeAdapter`.
- HL funding rates: `info.meta_and_asset_ctxs()` (NOT `info.meta()`); parallel arrays.
- Fees: `CalculatePlatformSpotFee` (HL 0.035%, RH 0%, BinanceUS 0.1%); `CalculatePlatformFuturesFee`.
- Position ownership: `Position.OwnerStrategyID`; `syncHyperliquidAccountPositions` reconciles owner only. **Shared-coin reconcile:** `reconcileHyperliquidAccountPositions` non-destructive for ambiguous gaps; three unambiguous-close detectors: (1) full flat (qty≈0 → close all peers); (2) SL-owner partial (residual matches peer sum → close owner); (3) TP partial fill → `recordPerpsExternalPartialCloseWithFillFee`. Real fee via `buildCachedHyperliquidReconcileFillResolver`; failure → modeled. `isHLLiveReconcilable`: HL+live+perps|manual. **SL attribution gate:** `hlReconcileSLFillConfirmed(lookup,useFillFee,stopLossOID)` — exact OID + `FilledQty>1e-9` — required by Detectors 1, 2, sole-owner vanish, and `hlAttemptCloseFromTPFills`; unconfirmed → `hl_sync_external`. **Owner DM on TP/SL fill:** `ProtectionFillAlert` appended to `pendingAlerts` under `mu`; `notifyProtectionFill` flushes after unlock. Toggle: `notify_tp_sl_fills` (`*bool`, nil→on). **Sole-owner TP attribution:** `tryBookSoleOwnerTPFill` at `HLFillLookup.Px`; `stampSoleOwnerRecoveryTierConsumed` prevents re-booking. **Full-close requires ALL `TPOIDs[i]==0`** — active tier defers to SL branch.
- **State is SQLite-only** (`scheduler/state.db`, `cfg.DBFile`). Trades persist via `RecordTrade` → `StateDB.InsertTrade`. `ClosedPositions` flushed by `SaveState`. Leaderboard on-demand (`BuildLeaderboardMessages`).
- **Trades schema:** `is_close`/`realized_pnl` flag close legs; `stampOpenTradeFromPosition` stamps close-leg `regime`/`entry_atr`/`stop_loss_trigger_px`. **Open-trade snapshot:** `recordPositionOpen` single INSERT; `DeferredOpen` variants run `runHyperliquidProtectionSync` before `RecordTrade` so OIDs land same-cycle. `LifetimeTradeStatsAll`: `#T` counts opens; W/L grouped by `(strategy_id, position_id)`.
- **Live execute fills:** `--execute` scripts emit `fillFee`+order IDs → `Trade.Fees`/`Trade.ExchangeOrderID`. **HL real fee:** `lookup_fill_fee_by_oid`; failure → modeled. Historical `exchange_fee=0` → `go-trader backfill hl-fees`. **HL `closedPnl` is gross** (`HLFillLookup.ClosedPnLGross`) — never wire into `Trade.RealizedPnL`; compute locally.
- **HL kill-switch fill accounting:** shared-coin fills split by virtual qty at snapshot time; snapshot under RLock BEFORE close mutates. `hyperliquidKillSwitchFillShare` fails closed when not a peer.
- **Trade.Side on close legs:** buy/sell semantics, not `"close"`. `closeTradeSide`: short→`"buy"`, else `"sell"`. `optionCloseTradeSide`: sell→`"buy"`, else `"sell"`.
- **Close-leg trade-alert `Source:` line:** close legs add `Source: <exchange SL|exchange TP{n}|close-strategy exit|external|circuit breaker|paper SL|paper trailing SL|trailing SL>` from `Trade.Details` prefix. `recordPerpsStopLossClose` writes prefix via `stopLossCloseDetailsPrefix(reason)`; `tradeAlertCloseSource` classifies. New reason → both helpers need a branch (paper-trailing substring check precedes plain trailing).
- **TradingView export:** when user says "export data to TradingView", first ask which strategy IDs (or all). Use `./go-trader export tradingview --strategy <id> --output <file>` or `--all --output <file>`. Sourced from SQLite trades; may require `tradingview_export.symbol_overrides` for unmapped platforms/symbols.
- **Map iteration:** ALWAYS `sort.Strings(keys)` before formatting operator-facing or test-asserted output — Go map iteration is randomized.
- `cfg.Discord.Channels` / `Telegram.Channels` / `DMChannels` are `map[string]string`; keys: `spot`, `options`, `<platform>`, `<platform>-paper`. `cfg.Discord.OwnerID` from `DISCORD_OWNER_ID` env (priority over config).
- `cfg.Discord.TradeAlertChannels`/`Telegram.TradeAlertChannels` (`map[string]string`, optional) — override channels for trade-fill alerts; falls back to `Channels`. SIGHUP-reloadable. A `stratType` key (e.g. `"perps"`) reroutes across all platforms — use platform key for per-platform.
- `cfg.SummaryFrequency map[string]string`: Go duration / alias (`hourly`, `daily`, `every`/`per_check`/`always`). `ShouldPostSummary` wall-clock throttles; `hasTrades=true` forces post. `lastSummaryPost` persisted in `app_state.last_summary_post` (use `cloneTimeMap`).
- `cfg.Correlation` (`Enabled=false`, `MaxConcentrationPct=60`, `MaxSameDirectionPct=75`). `cfg.AutoUpdate`: `off|daily|heartbeat`.
- **Regime detection:** `shared_tools/regime.py` 3-state labels (`trending_up`/`trending_down`/`ranging`) via ADX+DI; canonical set in `validRegimeLabels`. All 5 standard check scripts inject label; `check_options.py` defers. Go threads `--regime-{enabled,period,adx-threshold}` via `appendRegimeArgs`. Entry gate `regimeBlocksOpen` — close legs always pass. `AllowedRegimes` SIGHUP-compatible; `cfg.Regime` block requires restart.
- **Strategy registries:** open source of truth `shared_strategies/open/registry.py`; spot/futures shims build via `build_registry()`. Close: `shared_strategies/close/registry.py` with `evaluate(position,market,params)`. New open strategy: (1) `open/registry.py` + `PLATFORM_ORDER`, (2) `knownShortNames` + defaults in `init.go`, (3) `DEFAULT_PARAM_RANGES` in `backtest/optimizer.py`. Use `variants={"futures":{...}}`. Before refactoring registry: snapshot `--list-json` and `diff` after — Go `discoverStrategies` depends on byte-identical output.
- Strategy DSL gotchas: `apply_strategy(name,df,params)` merges config params UNDER runtime (runtime wins); `check_strategy.py` uses manual `sys.argv`, others argparse; `HTFFilter` not applied to options or `delta_neutral_funding`; `delta_neutral_funding` perps-only; perps vs futures `Position` both have `Multiplier>0`, only perps have `Leverage>0`.

## Build & Deploy
- **Update (operator + agents):** `bash scripts/update.sh --restart`. Atomic flow: preflight (hard-fails on missing `scheduler/config.json` / dirty build inputs) → `pull --ff-only` → `uv sync` → build `go-trader.new` → probe → atomic swap (`.prev` retention) → `restart`+verify → `do_rollback` on timeout (resets git to `pre_pull_sha`, re-syncs uv, restores `.prev`). Env: `STATUS_PORT`, `ACTIVE_TIMEOUT=30s`, `HEALTH_TIMEOUT=60s`. `applyUpgrade` calls same script. **Do not** rebuild Go in isolation — argv contract needs both sides at same SHA. **Go:** PATH `go` else `/opt/homebrew/bin/go` else `/usr/local/go/bin/go` (#764). **`--restart`:** stderr warning if unit `ExecStart` binary realpath ≠ repo `./go-trader` after swap. **Restart modes (#766):** `--restart-mode systemd` (default; `systemctl restart <unit>`, `MainPID` verify) or `--restart-mode signal` (Linux bare-process; SIGTERM old pid from `GO_TRADER_PIDFILE`/`./go-trader.pid`, respawn via `GO_TRADER_RUN_SH`/`./run.sh`, `/health`+PID verify + rollback parity). Generate `run.sh`: `bash scripts/create-run-sh.sh`. **Batch update (#766):** `bash scripts/update.sh --all --restart [--update-all-root <dir>]` iterates all `go-trader-*/` siblings. New signal-mode env: `GO_TRADER_RUN_SH`, `GO_TRADER_PIDFILE`, `GO_TRADER_SIGNAL_LOG`, `GO_TRADER_UPDATE_ALL_ROOT`.
- **Graceful shutdown:** `systemctl restart` drains side-effecting subprocesses up to `shutdownDrainCap=15s` then SIGKILLs; state-save / notifier flush / DB close run after. Unit `TimeoutStopSec=20`. Journal: `[shutdown] draining` / `State saved` / `Complete`. `/health` 503 while draining. Service-file change requires `daemon-reload`.
- Build (one-off): `go build -ldflags "-X main.Version=$(git describe --tags --always --dirty=-mod)" -o go-trader .` (without `-ldflags` summary shows `(dev)`). Rebuild before smoke-testing.
- Restart: `systemctl restart go-trader` (service-file changes: `daemon-reload` first). Config-only: `kill -HUP $(pgrep go-trader)`. Python changes pick up next cycle.
- **Startup compatibility probe:** binary invokes each unique check script with `--probe-only`; non-zero → log + owner DM + `os.Exit(1)`. Post-update probe failures → ensure `shared_scripts/` updated + rebuild.
- **Post-update protocol:** after `git pull` / auto-update, follow `SKILL.md` "Post-Update Agent Protocol". Diff `<running>..HEAD`, classify via SKILL.md Reference table, prompt per item. `runConfigMigrationDM` only covers `configFieldRegistry` (≤ v3).
- **Post-deploy smoke (#739):** after a release that changes the Python launcher, run `./go-trader --config scheduler/config.json --once` (or your deploy path) once to confirm startup probe and slow scripts still finish within timeouts.
